前端入门之JS数据类型

Posted on 2018-05-16

Javascript里的数据类型，共分为七种，js中的每一个值都属于某一种数据类型。

1.数字number：整数和小数。

2.字符串string：文本。

3.布尔值Boolean：true和false。

4.symbol类型：生成一个全局唯一的值。

5.null：空值。

6.undefined：未定义值。

7.对象object：值和键的组合，即为哈希表。包括数组及函数。

前六种为基本类型，最后一种为复杂类型。

1.数字类型：

js中可以用不同进制表示数字，通过不同的前缀表示不同进制：

十进制：没有前导0的数值。

八进制：有前缀0o或0O的数值，或者有前导0、且只用到0-7的八个阿拉伯数字的数值。

十六进制：有前缀0x或0X的数值

二进制：有前缀0b或者0B的数值

2.字符串：

用单引号或者双引号将文本括起来的类型。如果需要用特殊字符表示字符串可以用到转义字符\。

\0 ：null（\u0000）

\b ：后退键（\u0008）

\f ：换页符（\u000C）

\n ：换行符（\u000A）

\r ：回车键（\u000D）

\t ：制表符（\u0009）

\v ：垂直制表符（\u000B）

\’ ：单引号（\u0027）

\” ：双引号（\u0022）

\ ：反斜杠（\u005C）

字符串默认不能换行，如果长字符串必须分成多行，可以在每一行的尾部使用反斜杠。但是这种方式如果\后面有空格会报错而且不好查找错误。

我们还可以通过连接运算符（+）可以连接多个单行字符串，将长字符串拆成多行书写，输出的时候也是单行。

另一种es6的方法为用多行字符串来进行字符串的换行，这种方式在输出时会将换行也输出出来。

3.null与undefined

都表示空值，在if中都表示false，区别在于：

（规范）如果一个变量没有被赋值，那么这个变量的值就是 undefiend
（习俗）如果你想表示一个还没赋值的对象，就用 null。如果你想表示一个还没赋值的字符串/数字/布尔/symbol，就用 undefined（但是实际上你直接 var xxx 一下就行了，不用写 var xxx = undefined）
4.object类型

 对象就是一组“键值对”（key-value）的集合，是一种无序的复合数据集合。对象的另一种表现形式为哈希表，数组是一种特殊的对象。

对象的key值得引用可以通过xxx.key或者xxx['key']，但是当key为数字时只能通过xxx[1]来引用，此时虽然1没有加引号但是系统会默认给数字加引号。

对象的键名都是字符串，不加引号时系统会自动加上引号，但是当键名不符合标识符命名规则时，必须在键名上加上引号。

当通过xxx[key]来引用时，key未加引号，表明key为变量。

5.typeof 操作符

xxx 的类型 string number boolean symbol undefined null object function

typeof xxx ‘string’ number’ ‘boolean’ ‘symbol’ ‘undefined’ ‘object’ ‘object’ ‘function’

其中，比较特殊的为null的数据类型为null，但是typeof后为object，function的数据类型为object，但是typeof后的值为function。
